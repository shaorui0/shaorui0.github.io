## [](#关于学习新知识 "关于学习新知识")关于学习新知识

0.  从本质出发，寻找**模式**
    
    +   比如各种分布式中间件，各自的 cluster 架构是什么，为什么选择这种架构？本质还是从 一主多从、无主、多主中间选择一种。所以最终对比的是这三种 cluster 架构之间的区别。比如各种组件产生的分布式的问题（脑裂），怎么解决？本质也还是回到分布式的问题（而不是组件特定）。
    +   比如数据库的索引选择，本质还是 Hash(Consistent hashing) / B-Tree (family) / LSM-Tree 这些里面选择、改进
    +   比如抽象出来的设计模式。
    +   比如刷题也有一些[模式](https://zhuanlan.zhihu.com/p/161204634)
1.  抓主干，二八原则，其他靠实践中的零敲碎打
    
    > 我主张，在具备基础之后，学习任何新东西，都要抓住主线，突出重点。对于关键理论的学习，要集中精力，速战速决。而旁枝末节和非本质性的知识内容，完全可以留给实践去零敲碎打。
    
    > 原因是这样的，任何一个高级的知识内容，其中都只有一小部分是有思想创新、有重大影响的，而其它很多东西都是琐碎的、非本质的。因此，集中学习时必须把握住真正重要那部分，把其它东西留给实践。对于重点知识，只有集中学习其理论，才能确保体系性、连贯性、正确性，而对于那些旁枝末节，只有边干边学能够让你了解它们的真实价值是大是小，才能让你留下更生动的印象。如果你把精力用错了地方，比如用集中大块的时间来学习那些本来只需要查查手册就可以明白的小技巧，而对于真正重要的、思想性东西放在平时零敲碎打，那么肯定是事倍功半，甚至适得其反。 [https://blog.csdn.net/myan/article/details/5877305](https://blog.csdn.net/myan/article/details/5877305)
    
2.  学知识符合金字塔原则。先看架构（从书 -> 章节 -> 小节）。
    
3.  主动，从问题出发
    
    +   “WWWH”，常规问题，一般用于学习新概念
        +   Why? 为什么产生这个东西？ -> 它改进了什么？带来了什么新的问题/挑战？ -> 还有其他东西对它进一步改进吗？ -> trade off 是什么？各自的场合时怎样的？
    +   针对知识点的具体问题
        +   有兴趣的点先看，**以点到面**而不是瀑布式的全盘学习（学了忘等于没学且挤占了其他时间）
4.  输出
    
    +   写 Blog（本质是费曼技巧，讲给别人听）
        +   逻辑链要连贯，知道一个东西是怎么来的
    +   写各种 Demo 来学习
    +   构造复杂系统，复杂系统需要用到什么东西，需要学习什么？像点亮地图一样去学习。
